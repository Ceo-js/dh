Ciclo For

Ciclo For

Nós humanos somos maravilhosos! Criamos coisas fantásticas do nada. Entre tantas maravilhas, sentimos o que nos rodeia e agimos com base nesses sentidos. Mas, se existe algo que os humanos não são bons é em realizar tarefas repetitivas. Para isso, criamos as máquinas! Em nosso caso específico, programamos máquinas. E quando se trata de mandar uma máquina repetir uma ação, temos uma estrutura imbatível: o for!

A comunicação entre seres humanos e máquinas nem sempre é muito fácil. Podemos entender que programar é a forma que nós, seres pensantes, temos para dizer a uma máquina o que ela deve fazer. Pois bem, para que ela entenda a nossa intenção, precisamos ser diretos, explicando só e exatamente o que ela precisa fazer. No caso, para descrever uma repetição utilizamos três informações:

  • De onde começar;

  • O tamanho do passo;

  • A condição de parada.

Sim! Podemos fazer uma analogia com uma caminhada sobre uma linha de números! Cada passo na linha representa uma repetição. Se quisermos dizer para a máquina que ela deve sair do número 0 ao número 20 andando de 1 em 1, temos as seguintes informações:

  • De onde começar: 0;

  • Tamanho do passo: 1;

  • Condição de parada: posição deve ser menor que 21.

Em Javascript, ficaria assim:

for(let i = 0; i < 21; i = i + 1){
// Uma ou mais ações a realizar
}

No trecho de código acima, let i = 0 estabelece o valor inicial da variável i (no caso, zero). Essa variável existe para marcar a posição na caminhada. O trecho i < 21 define a condição de parada. Enquanto i for menor que 21, as ações a repetir serão realizadas e mais um passo na linha dos números será dado. Por fim, i = i + 1 determina o tamanho do passo: no caso 1. Muito complicado? Em Javascript, as funções são um tipo de dado e podemos armazená-las dentro de variáveis ou recebê-las como argumento de uma função (como acontece com os dados numéricos, arrays ou qualquer outro). Isso nos permite fazer coisas interessantes e poderosas, podendo aproveitar, por exemplo, o uso de callbacks.

Callbacks permitem passar uma função como parâmetro (argumento) de outra e por consequência executá-la. Mas… você sabe como isso acontece?

No vídeo abaixo vamos descobrir como isso é possível.

Callbacks

Exercício 1

Vamos criar três funções e ver como iniciar a trabalhar com os callbacks.

  • A primeira função se chamará dobro que receberá um número e retornará o dobro.

  • A segunda função se chamará triplo que recebera um número e retornará o triplo.

  • A terceira função será chamada de aplicar e receberá um valor e o nome de uma função como parâmetro. Ela retornará o valor da aplicação da referida função ao valor recebido.

Exemplo:

  • aplicar(2, dobro); // 4

  • aplicar(4, triplo); // 12

function dobro(numero){return numero * 2}
function triplo(numero){return numero * 3}
function aplicar(valor, funcao){
return funcao (valor)
}

Exercício 2

Agora vamos ver como podemos criar uma função que funcione como uma calculadora. Começaremos com algo simples.

Criaremos quatro funções que executarão as principais operações de uma calculadora: seu trabalho será definir as funções somar, subtrair, multiplicar e dividir. Cada uma delas receberá dois parâmetros e deve executar a operação matemática correspondente. Por exemplo, a função somar deve retornar a soma de ambos os parâmetros, a função subtrair a subtração de ambos os parâmetros e assim por diante

Agora, com as funções já definidas, criaremos a função calculadora. Esta função receberá dois parâmetros numéricos e o nome de uma função, que será nosso callback.

function somar( numero1, numero2 ) {
return numero1 + numero2;
}

function subtrair( numero1, numero2 ) {
return numero1 - numero2;
}

function multiplicar( numero1, numero2 ){
return numero1 * numero2;
}

function calculadora(numero1, numero2, funcao){
return funcao(numero1, numero2)
}

function dividir( numero1, numero2 ){
return numero1 / numero2;
}

Exercício 3

Sua tarefa agora é criar duas funções.

A primeira função deverá se chamar adicionarHttp , a mesma, receberá por parâmetro uma URL que será concatenada com a String "http://" no começo da URL e em seguida retornará esse valor. A segunda função se chamará processar e receberá uma lista de sites web e uma função. A função processar retornará um array contendo todos os resultados aplicando a função que foi passada.

function adicionarHttp(url) {
return "http://" + url
}

function processar(lista, callback){
return lista.map( i => callback( i ) );
}

Métodos de Arrays II

Métodos de Arrays II

O JavaScript nos fornece vários métodos (funções) de arrays, proporcionando uma gama de ferramentas para trabalhar com eles.

Antes de conhecer estes métodos, é necessário entender a definição de callback. Como acabamos de descobrir, as funções podem receber um ou mais parâmetros. Estes podem ser, entre outros: um número, uma string, um booleano ou também outra função. E a estas funções passadas como parâmetro de outra que chamamos de callback.

E por que estamos dizendo tudo isso? Porque, diferente dos primeiros métodos de array que você viu há pouco tempo, esses precisam de callbacks, funções que serão executadas dentro dos métodos de arrays.

Parece confuso? Então vamos entender melhor esses métodos!

Objeto Date

Objeto Date

Você já notou que alguns sites mostram a data e a hora do momento exato em que o visitamos. Mas como é possível fazer isso? Será preciso fazer download de alguma aplicação? Nada disso! O Javascript tem, nativamente, uma maneira de acessar essas informações pelo objeto date.

Para continuarmos, é importante lembrarmos que um objeto é a representação em código de um elemento da vida real, assim sendo conseguimos informações detalhadas além de um simples horário, tais como: mês, ano, milisegundos. Vem com a gente entender mais sobre o objeto date.

Objeto date
