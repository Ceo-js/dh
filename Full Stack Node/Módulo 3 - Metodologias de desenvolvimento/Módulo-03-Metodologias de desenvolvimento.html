O que é GIT?

Git é um sistema de controle de versão que registra as mudanças feitas em um arquivo ou conjunto de arquivos ao longo do tempo, possibilitando recuperar ou acessar versões específicas quando desejarmos. Além disso, é ideal para desenvolver projetos onde diversas pessoas podem contribuir simultaneamente, buscando evitar que suas alterações sejam sobrescritas.

O que é GitHub?

Mas o que seria o GitHub? É a mesma coisa que o Git?

Repositórios locais

Um repositório nada mais é do que uma pasta onde estão todos os arquivos do nosso projeto, incluindo nosso versionamento. Neste caso, o nosso repositório vai ser armazenado de forma local em nossa máquina, impossibilitando que outras pessoas tenham acesso.

Repositórios remotos

Um repositório remoto é criado em um local de hospedagem de repositórios Git, neste caso o GitHub, onde podemos armazenar uma cópia do nosso projeto na nuvem.

Usuário e senha? De onde?

Quando executamos os comandos do vídeo pela primeira vez, você pode ter uma certa dificuldade.

Como estamos utilizando o modo HTTPS, o GitHub precisa saber quem é você antes de aceitar que os commits feitos no seu repositório local sejam adicionados ao repositório remoto.

Tá bom. Mas pra que serve a configuração que eu fiz há duas aulas atrás? Não é pra isso?

Infelizmente, a resposta é não.

Ele serve, sim, para identificar quem fez o commit, sendo que se você não fizer essa configuração, na hora de confirmar os arquivos, o git emitirá um erro.

Mas o que vai para o GitHub é um "pacotinho", então ele espera que você se identifique. Senão, qualquer um poderia enviar código para repositórios que não são seus, o que seria um problemão!

Então, na primeira vez que você enviar commits para o GitHub, usando o HTTPS, ele vai te exigir um usuário e senha, que são os mesmos que você configurou quando criou sua conta no GitHub \o/.

Depois disso, aquele repositório não vai mais pedir senha ao enviar commits.

Treine bastante o uso de senha e, quando estiver confortável, pesquise sobre chaves SSH, para evitar digitar senha a todo momento. Esse é o modo mais profissional e o mais usado no mercado.

Com as chaves SSH, você pode trocar de senha tranquilamente, sem dor de cabeça com todos os repositórios que você teria que trocar.

A documentação do GitHub sobre chaves SSH é excelente para começar sua pesquisa. Recomendamos a leitura.
