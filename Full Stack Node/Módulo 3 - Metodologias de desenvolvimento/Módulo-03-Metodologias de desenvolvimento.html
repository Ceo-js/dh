Introdução ao GIT

Você já imaginou como pode trabalhar sem precisar ficar criando arquivos para cada versão do projeto e ter acesso ao histórico das suas alterações? Isso é possível através do versionamento de código.

No vídeo abaixo, iremos conhecer mais sobre uma ferramenta fundamental que nos possibilita gerenciar nossos projetos e suas diversas versões de forma rápida e fácil.

Vamos lá?

O que é GIT?

Git é um sistema de controle de versão que registra as mudanças feitas em um arquivo ou conjunto de arquivos ao longo do tempo, possibilitando recuperar ou acessar versões específicas quando desejarmos. Além disso, é ideal para desenvolver projetos onde diversas pessoas podem contribuir simultaneamente, buscando evitar que suas alterações sejam sobrescritas.

O que é GitHub?

Mas o que seria o GitHub? É a mesma coisa que o Git?

Você já imaginou como trabalhar em um ambiente profissional de forma colaborativa? De que maneira poderíamos compartilhar nossos documentos com nossa equipe de forma prática e de qualquer lugar precisando apenas de um acesso à internet? Podemos pensar em algumas opções como arquivos zip, Google Drive e outros tantos existentes.

Apesar das ferramentas acima serem boas, elas não são necessariamente dinâmicas ou até mesmo adequadas quando estamos dentro de um ambiente de desenvolvimento. Mas qual seria a ferramenta mais adequada para compartilhar nossos códigos?

Anteriormente, vimos sobre o Git e como ele funciona em relação ao gerenciamento de versões do nosso projeto, agora vamos lhe ensinar como acessar seu repositório na nuvem, onde você poderá baixá-lo e modificá-lo em qualquer computador do MUNDO.

Vamos lá?

O GitHub é uma plataforma criada para hospedar e controlar versões de projetos de programação na nuvem de forma gratuita, ou seja, você pode usá-lo para guardar seus projetos pessoais ou open source (projetos abertos a todos), possibilitando uma colaboração maior entre equipes de uma forma eficiente.

Git e GitHub

Muito bem! Embora eles tenham um nome semelhante, git é a tecnologia que mantém o sistema de versão. O GitHub é um provedor de serviços. Existem muitos outros como o GitLab ou o Bitbucket. Convido você a acessar os sites deles e ver o que eles oferecem.

Como criar seu primeiro Repositório

Repositórios

Repositórios locais

Um repositório nada mais é do que uma pasta onde estão todos os arquivos do nosso projeto, incluindo nosso versionamento. Neste caso, o nosso repositório vai ser armazenado de forma local em nossa máquina, impossibilitando que outras pessoas tenham acesso.

Repositórios remotos

Um repositório remoto é criado em um local de hospedagem de repositórios Git, neste caso o GitHub, onde podemos armazenar uma cópia do nosso projeto na nuvem.
