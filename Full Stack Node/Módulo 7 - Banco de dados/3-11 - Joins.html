Joins

Imagine o seguinte cenário: você tem uma tabela que armazena os dados de uma pessoa, e sabe que essa tabela, por sua vez, está ligada à outra tabela que armazena as imagens que uma pessoa posta. Mas, nesta última tabela, o único dado que se tem para identificar a pessoa dona dessa imagem é o ID dela, ou seja, o ID que corresponde a essa pessoa na tabela de pessoas.

Sendo assim, como poderíamos trazer, em uma só consulta, as imagens que pertencem ao ID 5, por exemplo? Em outras palavras, como é possível trazer todas as imagens que pertencem a uma pessoa se essa informação está presente em tabela?

Pois bem, para alcançar isso, o MySQL fornece uma ferramenta chamada de JOINS. Os joins (ou junções), tal qual seu nome indica, não são nada mais que uniões entre tabelas diferentes, que têm algum tipo de relação entre si.

Agora, com isso em mente, vamos ver como funciona no próximo vídeo.

Questionário Dia Mundial do Rock

  1. É a semana do rock ao nível mundial e nosso empresário da Musimundos nos pede para criar banners com músicas do gênero Rock. Faça uma consulta ao nosso banco de dados para retornar os nomes das músicas que possuem o gênero rock.

    Lembrando que o ID do gênero Rock é 6.

SELECT musica.nome, genero.nome FROM musica INNER JOIN genero ON musica.genero_id= genero.id WHERE genero.id = 6;
