O que vamos ver nessa aula

Até agora, te apresentamos ao mundo dos bancos de dados relacionais, fazendo as coisas na mão com MySQL. Agora, é chegado o momento de usar todos esses conceitos que você adquiriu e implementá-los diretamente em um projeto com Express.

No entanto, para que o trabalho com MySQL e Express seja muito mais simples, vamos te mostrar uma ferramenta magnífica chamada Sequelize, que é um ORM para Express e que manipula bancos de dados relacionais.

Mas, o que é um ORM? Pois bem, um ORM é o que conhecemos como Object Relational Mapping, que é uma ferramenta de programação que permite "transformar" as tabelas de um banco de dados em uma série de entidades que simplificam as tarefas básicas de acesso aos dados. Essas entidades, quando recebidas em nosso programa, tem um formato de Objeto Literal, e como desde muito tempo a linguagem mais utilizada para trabalhar com banco de dados é o SQL, esses tipos de ferramenta já tem bastante suporte e são muito bem recebidas por parte das pessoas que trabalham na área de desenvolvimento e programação.

Para que fique um pouco mais fácil de entender o que faz um ORM, este diagrama talvez te ajude.

Em uma tabela do MySQL:

id name price stock
1 Nike shoes 123.45 35
2 Reebok sneakers 110.34 45

Depois de passar pelo ORM

[ { id: 1,
name: 'Nike shoes',
price: 123.45,
stock: 35 },
{ id: 3,
name: 'Reebok sneakers',
price: 110.34,
stock: 45 }, ]

Assim, fica mais fácil de entender o que é um ORM. O que você vai ver nesta aula é:

  • Como instalar e começar a usar o ORM Sequelize;

  • O que são os modelos dentro do ORM;

  • Quais são os métodos do Sequelize que permitem fazer consultas do tipo: trazer todos, encontrar um, encontrar um por sua chave primária;

  • Como implementar operação de filtros tipo where, order by e limit.
