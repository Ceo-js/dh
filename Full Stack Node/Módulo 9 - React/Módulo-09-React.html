Introdução ao React

É uma biblioteca open source em Javascript para criar interfaces aos usuários. Assim, o tempo de carregamento das páginas é melhor, porque o React JS usa componentes que podem ser criados somente uma vez, e depois reutilizá-los quando for necessário mostrar um elemento no DOM.

O React viu a luz do dia em março de 2013 e foi criado por Jordan Walke, Engenheiro de Software no Facebook. A primeira implementação foi na timeline do Facebook, quando buscavam uma forma rápida e simples de carregar os conteúdos na rede social.

Hoje em dia, as maiores empresas do mundo usam o React para criar seus projetos, como Instagram, Twitter, Netflix, AirBnB, Pinterest, Trello… A lista é enorme. E a excelente notícia é que ele é simples de aprender, tem muitas vagas que pedem essa tecnologia e está em constante evolução.

Introdução aos componentes

Talvez uma das palavras mais ouvidas ao entrar no mundo de React seja "componente". Mas o que é um componente?

Os componentes são peças funcionais e fundamentais da aplicação, pois nos permitirão separar as diferentes partes que compõem a estrutura de um site em pequenas peças independentes e reutilizáveis. Estes são projetados para trabalhar isoladamente, mas para fazer parte de um "todo".

Tecnicamente, os componentes são funções JavaScript nativas que são executadas sempre que necessário. Como qualquer função, eles podem ser vinculados, ou não, a uma série de argumentos que nos permitem fazer que esse bloco de código possa ser reutilizável, estabelecendo que gera partes da interface do usuário com informações realmente dinâmicas.

JSX

É uma extensão JavaScript que serve para gerar blocos de código HTML, mas com instruções JavaScript.

Em si, JSX é muito semelhante ao HTML, mas com certas peculiaridades que o tornam diferente. Isso pode ser usado única e exclusivamente dentro do ambiente React.

Mas se ele se parece tanto com HTML, por que não usamos apenas HTML?

React propõe esse formato, pois, está ciente de que ao gerar a estrutura interna de um componente é muito certo de que grande parte desse código será dinâmico e reutilizável. É aqui que o JSX se torna vitalmente importante.

Para dar um exemplo, um simples bloco de código no JSX será assim:

<div> <h1>Ola! {user}!</h1> <img src="{user.avatar}">; <h2>Que bom ver você de volta.</h2> </div>

Como podemos ver, JSX não é tão distante do HTML, até se parece um pouco com EJS.

Então, vamos olhar com muito mais detalhes sobre esta nova, mas simples, forma de escrita.

Componentes statefull

Componentes statefull

Componentes Stateful ou com o estado, são aqueles que têm informação interna que, no seu tempo, ela pode ser devidamente modificada por esses componentes.

Este tipo de componente é altamente funcional já que nos permite deixar de depender de props para trabalhar com informação dinâmica.

No entanto, não devemos pensar que agora todos os nossos componentes devam ser "de estado", uma vez que esta decisão dependerá do contexto em que o referido componente é usado.

Ciclo de vida

Ciclo de vida de um componente

Certamente já pedimos comida em casa (ou entrega) de alguns desses aplicativos que estão na moda nos dias de hoje. Como vimos, depois que o pedido de comida é enviado, ele entra em um processo de várias etapas.

Por exemplo, a primeira coisa que acontece é que o aplicativo indica que está enviando o pedido para o restaurante. Mais tarde, quando o restaurante confirma, o que acontece é que a comida entra no processo de "preparação". Depois de pronta, passa para a fase de "nas mãos do entregador" e por fim a comida chega à fase de "entregue" para encerrar todo o processo.

O acima exposto nada mais é do que o ciclo de vida de um pedido. Do ponto inicial onde o pedido foi despachado, até o ponto final onde a comida chega à mesa.

Como neste cenário, algo muito semelhante acontece com os componentes. O React é responsável por fazê-los passar por três etapas claras, que são evidenciadas quando o componente:

  • É mostrado na tela (montagem).

  • Ele sofre alterações (atualização).

  • Deixe-o aparecer na tela (desmontagem).

O React chama todo esse processo de "ciclo de vida" e é algo muito importante de entender, pois ao longo dessas três etapas poderemos gerar diferentes funcionalidades se quisermos.
