Até agora, tudo o que construímos no React teve que ser renderizado no mesmo componente, mas se quiséssemos mostrar outro conteúdo, como poderíamos fazer isso?

O que veremos a seguir nos ajudará nesse sentido. A biblioteca React Router nos dá a possibilidade de renderizar um ou outro componente dependendo do caminho que acessamos na barra de endereço do navegador.

Ao final desta aula seremos capazes de:

Embora o React não precise da biblioteca React Router para funcionar, é importante esclarecer que esta é uma das bibliotecas mais comumente usadas no ambiente de desenvolvimento de aplicativos front-end com React. Portanto, saber como funciona e como implementá-lo é um ponto positivo fundamental.

SPA

SPA, o que é isso?

SPA ou também chamado de Single Page Application é um aplicativo da web no qual há apenas um único ponto de entrada, geralmente um arquivo index.html.

Em um SPA não há outro arquivo HTML que possa ser acessado separadamente, pois todo o conteúdo do aplicativo será carregado e processado no mesmo arquivo index.html.

Dentro deste contexto, mesmo que tenhamos apenas um arquivo, ainda temos a possibilidade de termos várias views que serão trocadas em sua exibição, produzindo assim o efeito de que existem várias "páginas" ou "arquivos" de conteúdo dentro da aplicação, quando a realidade é que tudo está sendo mostrado a partir de um único arquivo.

Um SPA oferece, por sua vez, uma experiência de usuário bastante agradável e fluida. Por não ter que fazer upload de um arquivo diferente, o upload do conteúdo será muito mais rápido. E é aqui que o React Virtual DOM assume um papel real, graças à sua existência será possível identificar quais os conteúdos da view que devem ser alterados e quais os que devem ser mantidos.

O conceito de SPA é relativamente novo e pode ser implementado graças a bibliotecas como o React. E faz ainda mais sentido ter roteadores de aplicativos que permitem escolher dinamicamente qual componente deve ser exibido na tela de acordo com o caminho na barra de aplicativos do navegador.

Como trabalhar com rotas parametrizadas

Como trabalhar com rotas parametrizadas

Como em um ambiente Express, dentro de React ( graças ao React Router) é possível gerar rotas parametrizadas para que com elas possamos carregar o mesmo componente, mas de forma dinâmica.

De forma dinâmica? Como?

Vamos imaginar que temos um componente que possui uma matriz de produtos dentro e que queremos mostrar um produto específico dependendo do id que é passado para nós através do caminho na barra de endereços do navegador.

Certamente, neste cenário, desejamos encontrar uma forma de capturar o id da rota e, a partir dela, escolhermos qual produto exibir.

Bem, é isso que o React Router nos permitirá fazer através das rotas parametrizadas. Mas não vamos conversar mais e ver em ação como esse processo pode ser realizado.

Desafio - React Router

Para permitir que nosso aplicativo React se comporte como um SPA real, teremos que instalar uma biblioteca adicional que nos permite gerenciar o sistema de roteamento de forma otimizada e inteligente. Qual deve ser o pacote a ser instalado?

Excelente. É essencial que você sempre tenha em mente que react-router-dom é um pacote que devemos instalar a partir do gerenciador de pacotes Node.js.

O que devemos fazer para implementar os componentes que o módulo react-router-dom traz?

Parabéns. Você deve ter em mente que no componente a ser criado, você pode importar os componentes react-router-dom que deseja usar.

Qual dos seguintes componentes do roteador geral é necessário para gerenciar as rotas de nosso projeto?

Muito bom. É como um wrapper necessário para gerenciar as rotas de nosso projeto.

Qual das seguintes linhas de código é correta para configurar qual componente deve ser processado com base no caminho que está sendo acessado?

Parabéns. Como você já sabe, o componente <Route /> tem dois atributos: path = ”” e component = {}. Ambos nos ajudam a configurar quais componentes devem ser renderizados com base no caminho que está sendo acessado. Além disso, o atributo exact permite que você identifique totalmente o componente exato que está sendo acessado.

Qual é o componente que nos permite hospedar todos os componentes <Route /> que definimos e também nos oferece uma maneira muito simples de controlar quando queremos acessar uma rota desconhecida?

Excelente. É claro que você tem muita clareza sobre as responsabilidades de cada um dos componentes do pacote react-router-dom.

Quando queremos começar a trabalhar com rotas parametrizadas, a primeira coisa que temos que fazer é defini-las. Com base nisso, qual das seguintes rotas parametrizadas está definida corretamente?

Perfeito. Como sabemos, com as rotas parametrizadas usam dois pontos (:) e o nome dos dados. É aqui que você vai perceber que todo o conhecimento já aprendido é muito útil, porque, como você pode ver, o comportamento de definição das rotas parametrizadas no React é semelhante ao que você viu no Node.js.

Como fazemos se quisermos armazenar ou acessar o valor que percorre por uma rota parametrizada?

Parabéns. Você nunca deve esquecer que o React Router nos fornece props que contêm informações sobre a rota que estabelecemos no componente <Route /> . Com os props, encontramos a .match,propriedade que nos oferece a propriedade .params. e contém as informações que trafegam pela rota parametrizada.
