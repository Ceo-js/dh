O que vamos ver na aula?

O que vamos ver nesta aula?

Neste ponto, já sabemos que dentro do React existem dois tipos de componentes: stateless e stateful. E certamente associamos cada um a um tipo particular de código. Por exemplo, se pensássemos em um componente statefull, provavelmente imaginaríamos um componente de classe. E embora isso esteja correto, veremos agora que componentes “tipo funções” também podem ter um estado.

O React, em suas últimas versões, implementou a possibilidade de atribuir estados, ciclos de vida e outros, a um componente funcional, pois dessa forma busca quebrar o gap diferencial na implementação entre componentes stateful e stateless.

Mas então, se um componente funcional pode ter estado, qual é o sentido de ter visto componentes de classe? Esta funcionalidade é relativamente nova, então alguns projetos React feitos com versões mais antigas podem ter componentes de classe. Nesse caso, devemos estar preparados para saber como lidar com eles.

Portanto, pontualmente ao longo desta aula aprenderemos sobre:

  • O que são hooks e quais benefícios eles proporcionam na criação de componentes.

  • Quais são os hooks mais importantes neste contexto.

  • Como trabalhar com os hooks: useState(), useEffect() e useRef().

  • Qual é a semelhança desses hook com relação às funcionalidades de um componente de classe.

Dito isso, vamos ao que interessa e começar esta aula.

Introdução a Hooks

Introdução a Hooks

A primeira coisa que precisamos esclarecer é que Hooks NÃO funcionam dentro de um componente de classe. É isso que chama a atenção neles, pois com eles poderemos gerar muitas funcionalidades tradicionalmente associadas a esses componentes, mas dentro de um componente funcional.

Então, o que são hooks?

Hooks são basicamente funções JavaScript puras que nos permitirão "ligar" recursos —como ciclo de vida e estado— a componentes não-classe, ou seja, a componentes funcionais. De acordo com a documentação oficial do React: "...Hooks não funcionam dentro de um componente de classe. Eles devem ser usados ??sem Classes."

Por que devemos usar hooks?

Como dissemos antes, os hooks nos dão a capacidade de ter o ciclo de vida e o estado disponível dentro dos componentes funcionais. Graças a isso, não precisaremos gerar um componente de classe para usar esses recursos.

Isso é tremendamente poderoso, pois, dessa forma, não importará mais em que tipo de componentes estamos, o importante aqui é saber que, se for necessário implementar um estado ou o ciclo de vida, será capaz de fazê-lo.

Além disso, a partir de agora, quando quisermos implementar o estado, não precisaremos mais fazer isso a partir do construtor da classe, como fazíamos antes. E também se quisermos atualizá-lo, teremos que usar o método setState(). Como os hooks são funções puras, poderemos implementar esses processos com mais facilidade.

Estas são as razões fundamentais pelas quais devemos usar hooks.

Desde quando os hooks são usados?

De acordo com a documentação oficial do React: "Hooks são uma nova adição desde a versão 16.8."

Isso significa que se nossa aplicação for feita com React, de fevereiro de 2019 em diante, dentro dela você terá a possibilidade de implementar os hooks sem nenhum problema.

Mas não vamos perder mais tempo com essa leitura e vamos ver como usar essa ótima funcionalidade que são os hooks.

useState()

Hooks - useState()

Este é o hook mais famoso e aquele que traz consigo a funcionalidade mais clara para entender o que são os hooks.

Vamos relembrar, por um momento, como o estado deve ser definido dentro de um componente de classe.

Para isso, foi necessário utilizar o construtor da classe e até mesmo chamar o super() para que tudo corresse conforme o esperado. Assim como quando quisermos atualizar o estado, foi necessário utilizar o setState(), pertencente à mesma classe.

Agora com os hooks para obter o mesmo resultado, será muito mais fácil, pois basta usar useState() e pronto. Isso é tudo.

Bem mais simples né?

Mas vamos ver isso em detalhes no próximo vídeo, para que possamos entender mais claramente como trabalhar com o estado do componente graças a este hook.

useEffect()

Hook - useEffect()

Este hook é tudo que precisamos para gerenciar o ciclo de vida de um componente.

Nesta única função temos a capacidade de controlar o que acontece quando o componente é montado, atualizado ou desmontado. É super funcional e relativamente fácil de usar.

Estando acostumado a fazer três coisas ao mesmo tempo, é importante entender como configurá-lo para fazer uma coisa ou outra, mas não será muito complicado. Justamente esse hook busca quebrar um pouco a lacuna de dificuldade que o ciclo de vida apresentou nos componentes de classe.

useRef()

Hook - useRef()

Este hook permite selecionar elementos DOM de forma fácil e simples, assim como fizemos com o querySelector de JavaScript Vanilla.

useRef() é realmente fácil de usar. Tudo o que precisamos fazer é executá-lo e armazenar seu valor de retorno em uma variável e, em seguida, usar essa variável na estrutura HTML do elemento que queremos capturar.
